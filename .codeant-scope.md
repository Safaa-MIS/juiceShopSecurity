# CodeAnt Scan Scope

The reviewer should scan the full repository from the project root.

## Do Not Exclude

Do not exclude these folders or files from review unless technically impossible:

- routes/
- lib/
- src/
- frontend/
- data/
- config/
- models/
- server files
- middleware files
- Dockerfile
- docker-compose files
- package.json
- package-lock.json
- swagger.yml
- OpenAPI files
- test files
- seed/demo files
- challenge/config files
- static files that are served publicly
- CI/CD workflow files

## Important

For this repository, test/demo/challenge files may still reflect real application behavior and should be reviewed.

Do not limit the review to only:

- server.ts
- package.json
- Dockerfile
- changed files
- dependency files

## Required Review Mode

Use full-repository analysis.

Enable all available scanners:

- SAST
- SCA
- Secrets
- Infrastructure / IaC
- Antipatterns
- SBOM / license
- AI code review
- Production-readiness review if available
- Performance review if available
